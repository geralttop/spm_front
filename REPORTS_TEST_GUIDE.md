# Тестирование системы жалоб

## Исправленные проблемы

### 1. Проблема с получением информации о пользователе
**Проблема**: В auth store не было поля `user`, из-за чего кнопки жалоб не отображались.

**Решение**: 
- Изменили логику получения информации о пользователе через `authApi.getProfile()`
- Добавили `currentUserId` в state компонентов
- Обновили условие `canReport` для использования `currentUserId`

### 2. Обновленные файлы:
- `spm_front/src/shared/ui/point-card.tsx` - исправлена логика отображения кнопки жалобы
- `spm_front/src/shared/ui/user-card.tsx` - исправлена логика отображения кнопки жалобы  
- `spm_front/src/widgets/Comments/Comments.tsx` - исправлен вызов loadCurrentUser

## Как протестировать

### 1. Подготовка
1. Убедитесь, что backend запущен (`npm run start:dev` в SPM_backend)
2. Убедитесь, что frontend запущен (`npm run dev` в spm_front)
3. Зарегистрируйтесь или войдите в систему

### 2. Тестирование жалоб на точки
1. Перейдите на главную страницу или в ленту
2. Найдите точку, созданную другим пользователем
3. Рядом с кнопкой сердечка (избранное) должна быть кнопка флага
4. Нажмите на кнопку флага
5. Должно открыться модальное окно с формой жалобы
6. Выберите причину и отправьте жалобу
7. Должно появиться уведомление об успешной отправке

### 3. Тестирование жалоб на комментарии
1. Откройте любую точку с комментариями
2. Нажмите "Показать комментарии"
3. Для комментариев других пользователей должна быть кнопка "Пожаловаться" с иконкой флага
4. Нажмите на кнопку
5. Заполните форму жалобы и отправьте

### 4. Тестирование жалоб на пользователей
1. Перейдите в поиск пользователей
2. Найдите любого пользователя
3. В карточке пользователя должна быть кнопка флага
4. Нажмите на неё и отправьте жалобу

### 5. Проверка ограничений
- Кнопки жалоб НЕ должны отображаться для собственного контента
- Кнопки НЕ должны отображаться для неавторизованных пользователей
- После отправки жалобы должно появляться уведомление

## Отладка

### Если кнопки не отображаются:
1. Проверьте, что пользователь авторизован
2. Откройте DevTools и проверьте консоль на ошибки
3. Проверьте, что `currentUserId` загружается правильно
4. Убедитесь, что это не ваш собственный контент

### Если модальное окно не открывается:
1. Проверьте консоль на ошибки импорта
2. Убедитесь, что ReportModal правильно экспортируется из shared/ui

### Если жалоба не отправляется:
1. Проверьте Network tab в DevTools
2. Убедитесь, что backend запущен и доступен
3. Проверьте, что JWT токен валиден

## Ожидаемое поведение

### Кнопки жалоб должны появляться:
- ✅ На точках других пользователей (флаг рядом с сердечком)
- ✅ На комментариях других пользователей (кнопка "Пожаловаться")
- ✅ В карточках других пользователей (кнопка флага)

### Кнопки НЕ должны появляться:
- ❌ На собственных точках
- ❌ На собственных комментариях  
- ❌ В собственной карточке пользователя
- ❌ Для неавторизованных пользователей

### После отправки жалобы:
- ✅ Модальное окно закрывается
- ✅ Появляется зеленое уведомление об успехе
- ✅ Форма сбрасывается для следующего использования

## API endpoints для проверки

Если нужно проверить, что жалобы доходят до сервера:

```bash
# Получить все жалобы (только для админов)
curl -X GET http://localhost:3000/admin/reports \
  -H "Authorization: Bearer ADMIN_JWT_TOKEN"

# Создать жалобу (для обычных пользователей)
curl -X POST http://localhost:3000/reports \
  -H "Authorization: Bearer USER_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "point",
    "reason": "spam", 
    "description": "Тестовая жалоба",
    "pointId": "point-uuid"
  }'
```
