# Добавление кнопок подписки/отписки в модальные окна

## Новая функциональность
В модальных окнах подписчиков и подписок теперь отображаются кнопки для управления подписками.

## Изменения

### 1. Новые состояния
- `followingStates: Record<number, boolean>` - отслеживает статус подписки на каждого пользователя
- `actionLoadingStates: Record<number, boolean>` - отслеживает состояние загрузки для каждой кнопки

### 2. Обновленные функции

#### handleShowFollowers()
- Загружает список подписчиков
- Для каждого подписчика проверяет, подписаны ли мы на него через `subscriptionsApi.getStats()`
- Инициализирует состояния подписок

#### handleShowFollowing()
- Загружает список подписок
- Инициализирует все состояния как `true` (мы подписаны на всех в этом списке)

#### handleFollowToggleInModal()
- Управляет подпиской/отпиской в модальных окнах
- Обновляет локальные состояния и статистику
- При отписке в модальном окне подписок удаляет пользователя из списка

### 3. Обновленные модальные окна

#### Модальное окно подписчиков
- Показывает кнопку "Подписаться" или "Отписаться" в зависимости от текущего статуса
- Кнопка не показывается для собственного профиля

#### Модальное окно подписок
- Показывает кнопку "Отписаться" для всех пользователей
- При отписке пользователь исчезает из списка
- Кнопка не показывается для собственного профиля

### 4. UX улучшения
- Клик по информации пользователя ведет на его профиль
- Клик по кнопке не вызывает переход на профиль (stopPropagation)
- Кнопки показывают состояние загрузки
- Статистика обновляется в реальном времени

## Поведение
- **В подписчиках**: можно подписаться на тех, на кого не подписан, или отписаться
- **В подписках**: можно только отписаться (при отписке пользователь исчезает из списка)
- Все изменения сразу отражаются в статистике подписок/подписчиков
