# Исправление проблемы с выбросом из аккаунта при переходе /user/[id] → /profile

## Проблема
При переходе на свой профиль через ссылку `/user/[id]`, пользователя перенаправляло на `/profile`, но затем сразу выбрасывало на страницу авторизации.

## Причины
1. **Race condition в AuthInitializer** - инициализация аутентификации могла не завершиться до API вызовов
2. **Агрессивная логика logout** - интерцептор сразу редиректил на /auth при любой ошибке 401
3. **Отсутствие проверки аутентификации** перед API вызовами на странице `/user/[id]`
4. **Синхронная checkAuth** - не могла обновить токен при необходимости

## Исправления

### 1. Улучшен AuthInitializer
- Убран timeout в 2 секунды
- Теперь ждет полной инициализации аутентификации

### 2. Улучшена логика API интерцептора
- Убрана агрессивная переадресация на /auth
- Ошибки обрабатываются на уровне компонентов
- Более мягкий подход к обработке ошибок refresh токена

### 3. Добавлена проверка аутентификации в /user/[id]
- Проверка `checkAuth()` перед API вызовами
- Повторная проверка при ошибках
- Более умная обработка ошибок

### 4. Улучшена функция checkAuth
- Теперь асинхронная
- Может автоматически обновлять токен
- Возвращает актуальное состояние аутентификации

### 5. Обновлены все компоненты
- Header теперь использует состояние аутентификации
- Главная страница и страница авторизации обновлены
- Все вызовы checkAuth теперь асинхронные

## Результат
Теперь при переходе на `/user/[id]` (свой профиль):
1. Проверяется аутентификация
2. При необходимости обновляется токен
3. Выполняется безопасный редирект на `/profile`
4. Пользователь остается авторизованным
