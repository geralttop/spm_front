# Поток авторизации в админке

## Как это работает

### 1. Хранение токенов

Приложение использует Zustand store для управления состоянием авторизации:
- Токен хранится в `localStorage` под ключом `auth-storage`
- Структура: `{ state: { accessToken, tokenExpiry, isAuthenticated } }`

### 2. Проверка доступа к админке

При переходе на `/admin`:

1. **Проверка авторизации** - используется `useAuthStore.checkAuth()`
2. **Получение профиля** - запрос к `/auth/profile` с токеном
3. **Проверка роли** - должна быть `role: 'admin'`
4. **Редирект** - если что-то не так, редирект на `/auth` или `/`

### 3. React Admin провайдеры

**authProvider:**
- Читает токен из `auth-storage`
- Декодирует JWT для получения данных пользователя
- Проверяет роль admin

**dataProvider:**
- Читает токен из `auth-storage`
- Добавляет `Authorization: Bearer <token>` ко всем запросам
- Работает с `/admin/*` эндпоинтами

### 4. Автоматическое обновление токена

- Access token живет 15 минут
- За 2 минуты до истечения автоматически обновляется
- Refresh token хранится в httpOnly cookie
- При истечении токена пользователь перенаправляется на `/auth`

## Важно

- Админка НЕ имеет собственной формы входа
- Вход происходит через основную страницу `/auth`
- После входа под админом в сайдбаре появляется пункт "Админка"
- Токены синхронизированы между основным приложением и админкой
