# Страница создания точек

## Новая функциональность
Создана полноценная страница для создания новых точек на карте с возможностью указания всех необходимых параметров.

## Структура данных (из бэкенда)

### Point Entity
- `id` - UUID точки
- `name` - название точки
- `description` - описание (опционально)
- `address` - адрес (опционально)
- `coords` - координаты в формате GeoJSON Point
- `createdAt` - дата создания
- `category` - связь с категорией
- `container` - связь с контейнером

### Category Entity
- `id` - числовой ID категории
- `name` - название категории
- `icon` - название иконки
- `color` - цвет метки на карте
- `author` - автор (null для системных категорий)

### Container Entity
- `id` - UUID контейнера
- `title` - название контейнера
- `description` - описание контейнера
- `createdAt` - дата создания
- `author` - автор контейнера

## API обновления

### Новые методы в pointsApi:
- `create(data: CreatePointRequest)` - создание новой точки

### Новые API:
- `categoriesApi.getAll()` - получение всех категорий
- `containersApi.getAll()` - получение всех контейнеров пользователя
- `containersApi.create(data)` - создание нового контейнера

### Типы данных:
- `CreatePointRequest` - данные для создания точки
- `Category` - структура категории
- `Container` - структура контейнера

## Страница создания точки (`/points/create`)

### Основные секции:

#### 1. Основная информация
- **Название точки** (обязательно, до 100 символов)
- **Описание** (опционально, до 1000 символов)

#### 2. Координаты
- **Долгота (lng)** - числовое поле с примером
- **Широта (lat)** - числовое поле с примером
- Примеры координат для Минска

#### 3. Категория
- Выпадающий список всех доступных категорий
- Обязательное поле

#### 4. Контейнер
- Выпадающий список контейнеров пользователя
- Возможность создать новый контейнер прямо на странице
- Обязательное поле

### Функциональность создания контейнера:
- Раскрывающаяся форма для создания нового контейнера
- Поля: название (обязательно), описание (опционально)
- Автоматический выбор созданного контейнера
- Обновление списка контейнеров после создания

### UX особенности:
- **Валидация формы** - проверка обязательных полей
- **Состояния загрузки** - индикаторы для всех асинхронных операций
- **Сообщения об ошибках/успехе** - информативная обратная связь
- **Автоматический редирект** - на профиль после успешного создания
- **Кнопка "Назад"** - возврат на предыдущую страницу

## Техническая реализация

### Состояния компонента:
- `loading` - загрузка начальных данных
- `creating` - процесс создания точки
- `categories` - список категорий
- `containers` - список контейнеров
- `formData` - данные формы
- `showCreateContainer` - показ формы создания контейнера
- `error/success` - сообщения пользователю

### Валидация:
- Проверка обязательных полей перед отправкой
- Валидация координат (числовые значения)
- Ограничения по длине текстовых полей

### Безопасность:
- Проверка аутентификации при загрузке страницы
- Редирект на страницу авторизации для неавторизованных пользователей
