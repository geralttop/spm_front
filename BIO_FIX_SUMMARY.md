# Исправление проблемы с отображением биографии после обновления страницы

## Проблема
Биография сохранялась в БД при редактировании профиля, но не отображалась после обновления страницы.

## Причина
Метод `getProfile` в контроллере возвращал данные из JWT токена (`req.user`), а не из базы данных. В JWT токене не было поля `bio`, поэтому биография не передавалась на фронтенд.

## Исправления

### 1. Обновлен метод getProfile в AuthController
- Теперь загружает актуальные данные пользователя из БД через `usersService.findById()`
- Возвращает полную структуру профиля включая `bio`

### 2. Обновлен UserProfileDto
- Добавлено поле `bio?: string`
- Приведена структура в соответствие с фактическим ответом API
- Добавлены поля `sub`, `userId`, `role` для совместимости с фронтендом

## Результат
Теперь при обновлении страницы профиля:
1. Метод `getProfile` загружает актуальные данные из БД
2. Биография корректно отображается
3. Все изменения профиля сохраняются и отображаются после перезагрузки

## Техническая информация
- Поле `bio` существует в сущности User как `@Column({ type: 'text', nullable: true })`
- Метод `findById` в UsersService возвращает все поля пользователя
- Фронтенд получает актуальные данные при каждом запросе профиля
