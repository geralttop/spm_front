# Исправление перенаправления на собственный профиль

## Проблема
При переходе на `/user/[id]` со своим собственным ID пользователь видел страницу как будто это другой пользователь, вместо перенаправления на `/profile`.

## Решение

### 1. Добавлена проверка в `/user/[id]/page.tsx`
При загрузке страницы:
- Получаем текущий профиль пользователя
- Сравниваем ID из URL с ID текущего пользователя
- Если совпадают → перенаправляем на `/profile`
- Если не совпадают → показываем профиль другого пользователя

### 2. Обновлены модальные окна подписчиков/подписок
В обоих страницах (`/profile` и `/user/[id]`):
- При клике на пользователя в списке проверяем, не наш ли это профиль
- Если наш → перенаправляем на `/profile`
- Если чужой → перенаправляем на `/user/[id]`

## Поведение

### На странице `/user/[id]`
```typescript
// Если userId === currentUserId
router.push("/profile"); // Перенаправление

// Если userId !== currentUserId
// Показываем профиль другого пользователя
```

### В модальных окнах
```typescript
// При клике на пользователя
if (userId === currentUserId) {
  router.push("/profile");
} else {
  router.push(`/user/${userId}`);
}
```

## Результат
- Пользователь не может просматривать свой профиль через `/user/[id]`
- Автоматическое перенаправление на `/profile` при попытке
- Корректная навигация из списков подписчиков/подписок
