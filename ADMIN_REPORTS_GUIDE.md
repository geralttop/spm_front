# Админка для управления жалобами

## ✅ Что добавлено

### 1. Новый ресурс "Жалобы" в админке
- **Список жалоб** с фильтрацией по статусу, типу и причине
- **Просмотр деталей** каждой жалобы
- **Редактирование** статуса и заметок администратора
- **Быстрые действия** прямо из списка

### 2. Функционал обработки жалоб
- **Удалить объект** - удаляет точку/комментарий, на который пожаловались
- **Заблокировать пользователя** - блокирует автора контента
- **Отклонить жалобу** - помечает жалобу как необоснованную

### 3. Статистика на Dashboard
- **Общее количество жалоб** в карточке статистики
- **Статус жалоб** - сколько ожидают, решены, отклонены
- **Диаграммы** - распределение по статусам и типам

## 🎯 Как использовать

### Доступ к жалобам
1. Войдите в админку как администратор
2. В боковом меню выберите "Жалобы"
3. Увидите список всех жалоб с возможностью фильтрации

### Обработка жалобы

#### Быстрые действия из списка:
- **🗑️ Удалить объект** - удаляет точку/комментарий
- **❌ Заблокировать** - блокирует пользователя
- **✅ Отклонить** - отклоняет жалобу

#### Детальная обработка:
1. Нажмите на жалобу для просмотра деталей
2. Изучите информацию о жалобщике и объекте жалобы
3. Используйте кнопки действий или перейдите к редактированию
4. Добавьте заметки администратора при необходимости

### Фильтрация жалоб
- **По статусу**: pending, reviewed, resolved, dismissed
- **По типу**: point, comment, user  
- **По причине**: spam, inappropriate, harassment, fake, other

- **Общее количество жалоб**
- **Жалобы ожидающие рассмотрения** (требуют внимания)

### Полезные метрики:

## 🔧 Технические детали

### Файлы:
- `src/features/admin/resources/reports.tsx` - компоненты админки для жалоб
- `src/features/admin/AdminApp.tsx` - добавлен ресурс жалоб
- `src/features/admin/dataProvider.ts` - поддержка действий с жалобами

### API endpoints:
- `GET /admin/reports` - список жалоб
- `GET /admin/reports/:id` - детали жалобы
- `PUT /admin/reports/:id` - обновление жалобы
- `POST /admin/reports/:id/resolve` - действия с жалобой
- `DELETE /admin/reports/:id` - удаление жалобы

### Действия с жалобами:
```json
{
  "action": "delete|block|dismiss",
  "adminNotes": "Заметки администратора"
}
```

## 📋 Workflow обработки жалоб

1. **Поступление жалобы** - пользователь отправляет жалобу
2. **Уведомление** - жалоба появляется в админке со статусом "pending"
3. **Рассмотрение** - админ изучает детали жалобы
4. **Принятие решения**:
   - Удалить контент (если нарушение подтверждено)
   - Заблокировать пользователя (при серьезных нарушениях)
   - Отклонить жалобу (если нарушения нет)
5. **Документирование** - добавление заметок администратора
6. **Завершение** - жалоба получает финальный статус

Теперь администраторы могут эффективно управлять жалобами и поддерживать порядок в системе!
