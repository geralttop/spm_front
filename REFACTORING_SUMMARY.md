## 🎨 Архитектурные улучшения

### Feature-Sliced Design

Проект следует принципам FSD:

```
src/
├── shared/           # Переиспользуемый код
│   ├── api/         # API клиенты
│   ├── ui/          # UI компоненты
│   ├── lib/         # Хуки и утилиты
│   └── config/      # Конфигурация
├── entities/        # Бизнес-сущности
├── features/        # Фичи приложения
└── widgets/         # Составные блоки
```

### Separation of Concerns

- **API слой**: изолирован в `src/shared/api/`
- **UI слой**: изолирован в `src/shared/ui/`
- **Бизнес-логика**: в хуках `src/shared/lib/hooks/`
- **Утилиты**: в `src/shared/lib/utils/`

### Single Responsibility

Каждый компонент/хук отвечает за одну задачу:
- `UserListModal` - только отображение списка
- `useFollowManagement` - только управление подписками
- `useUserModal` - только управление модальным окном
- `formatRelativeDate` - только форматирование даты
