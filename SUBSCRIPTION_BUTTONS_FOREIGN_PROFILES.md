# Добавление кнопок подписки в модальные окна чужих профилей

## Обновление
Теперь кнопки подписки/отписки отображаются не только в собственном профиле, но и при просмотре подписчиков/подписок чужих профилей.

## Изменения в `/user/[id]/page.tsx`

### 1. Новые состояния
- `followingStates: Record<number, boolean>` - отслеживает статус подписки на каждого пользователя
- `actionLoadingStates: Record<number, boolean>` - отслеживает состояние загрузки для каждой кнопки

### 2. Обновленные функции

#### handleShowFollowers()
- Загружает подписчиков просматриваемого профиля
- Для каждого подписчика проверяет, подписаны ли мы на него через `subscriptionsApi.getStats()`
- Инициализирует состояния подписок

#### handleShowFollowing()
- Загружает подписки просматриваемого профиля
- Для каждого пользователя проверяет, подписаны ли мы на него
- Инициализирует состояния подписок

#### handleFollowToggleInModal()
- Управляет подпиской/отпиской в модальных окнах
- Обновляет локальные состояния
- Работает независимо от основной статистики профиля

### 3. Обновленные модальные окна

#### Модальное окно подписчиков
- Показывает кнопки подписки/отписки для всех пользователей (кроме себя)
- Цвет кнопки зависит от текущего статуса подписки
- Клик по профилю ведет на страницу пользователя
- Клик по кнопке управляет подпиской

#### Модальное окно подписок
- Показывает кнопки подписки/отписки для всех пользователей (кроме себя)
- Позволяет подписаться на тех, на кого не подписан
- Позволяет отписаться от тех, на кого подписан

## Логика работы

### При просмотре чужого профиля:
1. Открываем модальное окно подписчиков/подписок
2. Для каждого пользователя проверяем наш статус подписки
3. Отображаем соответствующую кнопку (синюю "Подписаться" или красную "Отписаться")
4. При клике на кнопку меняем статус подписки
5. Кнопка обновляется в реальном времени

### Цветовая схема:
- 🔵 **Синяя кнопка**: "Подписаться" (не подписан)
- 🔴 **Красная кнопка**: "Отписаться" (подписан)

## UX улучшения
- Кнопки не показываются для собственного профиля
- Состояние загрузки для каждой кнопки индивидуально
- Клик по информации пользователя и клик по кнопке разделены
- Мгновенное обновление состояния кнопок после действия
