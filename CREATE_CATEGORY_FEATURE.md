# Добавление возможности создания категорий

## Новая функциональность
Добавлена возможность создавать новые категории прямо на странице создания точки, аналогично созданию контейнеров.

## Изменения в API

### Обновлен categoriesApi:
```typescript
categoriesApi.create(data: { 
  name: string; 
  icon?: string; 
  color?: string 
}): Promise<Category>
```

## Изменения на странице создания точки

### 1. Новые состояния
- `showCreateCategory` - показ/скрытие формы создания категории
- `newCategoryName` - название новой категории
- `newCategoryIcon` - иконка новой категории
- `newCategoryColor` - цвет новой категории (по умолчанию #3B82F6)
- `creatingCategory` - состояние загрузки при создании

### 2. Функция создания категории
```typescript
handleCreateCategory()
```
- Валидация обязательного поля (название)
- Создание категории через API
- Обновление списка категорий
- Автоматический выбор созданной категории
- Очистка формы и закрытие
- Показ сообщения об успехе

### 3. Обновленная секция категорий

#### Основные элементы:
- **Выпадающий список** существующих категорий
- **Кнопка "Создать новую категорию"** с иконкой Plus
- **Раскрывающаяся форма** для создания новой категории

#### Форма создания категории:
- **Название** (обязательно, до 50 символов)
- **Иконка** (опционально, до 30 символов)
- **Цвет** (color picker с отображением HEX значения)
- **Популярные иконки** - быстрый выбор из предустановленных вариантов

### 4. Популярные иконки
Добавлены кнопки для быстрого выбора популярных иконок:
- coffee, restaurant, park, shop
- hospital, school, bank, gas-station  
- hotel, museum

При клике на кнопку иконка автоматически подставляется в поле.

## UX улучшения

### Валидация:
- Проверка обязательного поля "название"
- Ограничения по длине текстовых полей
- Цвет в формате HEX (валидация на бэкенде)

### Состояния интерфейса:
- **Кнопка создания** блокируется при пустом названии
- **Индикатор загрузки** "Создание..." во время запроса
- **Сообщения об успехе/ошибке** с информативным текстом

### Автоматизация:
- **Автоматический выбор** созданной категории
- **Очистка формы** после успешного создания
- **Закрытие формы** после создания
- **Обновление списка** категорий в реальном времени

## Структура данных категории

### Поля:
- `id` - числовой идентификатор
- `name` - название категории
- `icon` - название иконки (опционально)
- `color` - цвет в формате HEX

### Пример создаваемой категории:
```json
{
  "name": "Кафе",
  "icon": "coffee",
  "color": "#8B5CF6"
}
```

## Безопасность
- Создаваемые категории привязываются к текущему пользователю
- Пользователь видит только свои категории + системные
- Валидация данных на бэкенде (DTO с декораторами)

## Будущие улучшения
- Предпросмотр иконки в реальном времени
- Больше предустановленных иконок
- Группировка иконок по категориям
- Импорт иконок из внешних библиотек
- Цветовые палитры для быстрого выбора
