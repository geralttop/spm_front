# Исправление проблемы с выходом при редиректе на профиль

## Проблема
При переходе на свой профиль через `/user/[id]` происходил редирект на `/profile`, но затем пользователя сразу выкидывало из аккаунта.

## Причина
1. На странице `/user/[id]` при обнаружении своего ID происходил `router.push("/profile")`
2. На странице `/profile` в `useEffect` сразу вызывался `checkAuth()`
3. `checkAuth()` проверял токен синхронно, но токен мог еще не быть обновлен через `initializeAuth`
4. Если `checkAuth()` возвращал `false`, происходил редирект на `/auth`

## Решение

### 1. Убрана преждевременная проверка `checkAuth()`
**Файлы:** 
- `spm_front/app/profile/page.tsx`
- `spm_front/app/user/[id]/page.tsx`

Вместо проверки `checkAuth()` перед запросом:
```typescript
// БЫЛО (неправильно)
if (!checkAuth()) {
  router.push("/auth");
  return;
}
const data = await authApi.getProfile();
```

Теперь сразу делаем запрос:
```typescript
// СТАЛО (правильно)
try {
  const data = await authApi.getProfile();
  // ... обработка данных
} catch (error) {
  // Только если запрос не удался, редиректим
  router.push("/auth");
}
```

### 2. Улучшен AuthInitializer
**Файл:** `spm_front/src/app/auth-initializer.tsx`

Добавлен:
- Таймаут 2 секунды для инициализации (не блокирует приложение надолго)
- Индикатор загрузки вместо пустого экрана
- Обработка ошибок инициализации

## Как это работает теперь

1. **При загрузке приложения:**
   - `AuthInitializer` пытается обновить токен (максимум 2 секунды)
   - Показывает "Loading..." во время инициализации
   - После инициализации рендерит приложение

2. **При переходе на `/user/[id]` со своим ID:**
   - Делается запрос `getProfile()` для получения текущего userId
   - Если ID совпадает → редирект на `/profile`
   - Интерцептор в `client.ts` автоматически добавляет/обновляет токен

3. **На странице `/profile`:**
   - Сразу делается запрос `getProfile()` без проверки `checkAuth()`
   - Если запрос успешен → показываем профиль
   - Если запрос не удался (401) → интерцептор пытается обновить токен
   - Если обновление не удалось → редирект на `/auth`

## Преимущества

- Нет race condition между `checkAuth()` и обновлением токена
- API запросы сами управляют аутентификацией через интерцепторы
- Более надежная работа при редиректах
- Лучший UX с индикатором загрузки

## Результат
Теперь при переходе на свой профиль через `/user/[id]` происходит корректный редирект на `/profile` без выхода из аккаунта.
